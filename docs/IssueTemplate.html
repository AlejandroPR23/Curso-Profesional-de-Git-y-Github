Crear un Issue Template 📝

Para mejorar la gestión de problemas en tu repositorio de GitHub, puedes crear una plantilla para reportar errores. Aquí te explico cómo hacerlo:

Pasos para crear un Issue Template

📂 Crear la estructura de carpetas

  • mkdir .github - Crea la carpeta .github en tu repositorio local.
  • mkdir .github/ISSUE_TEMPLATE - Dentro de .github, crea la carpeta ISSUE_TEMPLATE.
  • touch .github/ISSUE_TEMPLATE/bug_report.md - Crea un archivo llamado bug_report.md para la plantilla.

📄 Agregar la plantilla de reporte de errores

En bug_report.md, agrega el siguiente contenido para facilitar la creación de reportes de errores:


# Reporte de Error

**Descripción del error:**
[Describe brevemente el error]

**Pasos para reproducir:**
1. [Paso 1]
2. [Paso 2]
3. [Paso 3]

**Resultado esperado:**
[Describe el resultado que esperabas]

**Resultado actual:**
[Describe lo que realmente sucedió]

**Información adicional:**
[Agrega cualquier otra información relevante, como capturas de pantalla o detalles del entorno]

🔍 Encontrar la plantilla en GitHub

Ve a la pestaña de Issues en tu repositorio en GitHub, y encontrarás la plantilla disponible. Puedes agregar más plantillas para otros tipos de problemas creando archivos adicionales en la carpeta ISSUE_TEMPLATE.
